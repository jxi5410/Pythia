## What to Build

### 1. src/pythia_live/companion.py — The Conversational AI Engine

The brain. NOT a command parser. A conversational agent.

Personality: Sharp, concise, trader-fluent. No corporate speak. Confident but precise. States confidence levels. Uses market shorthand (bps, pts, vol). Never says "I think" — says "my data shows" or "historical pattern suggests". Slightly competitive edge: "you are seeing this before it hits the wire"

Core class:
- PythiaCompanion with respond(), generate_proactive_alert(), morning_briefing(), weekly_digest()
- Takes natural language input, returns conversational response
- Maintains per-user context (recent messages, watchlist, preferences)
- No external LLM needed for v1 — rule-based NLU with template responses + dynamic data

Intent parsing from message:
- "whats moving" / "anything happening" / "status" -> active confluence events + top movers
- "fed" / "rates" / "fomc" -> filter for fed_rate category
- "china" / "tariffs" -> filter for china_macro/tariffs
- "crypto" / "bitcoin" / "btc" -> filter for crypto
- "why did X move" / "what caused" -> causal attribution
- "watch X" / "alert me on X" -> add to watchlist + set alert
- "regime" / "macro" / "whats the vibe" -> regime state
- "proof" / "track record" / "how accurate" -> track record stats
- "detail X" / "tell me about X" -> contract detail
- "morning" / "briefing" -> morning briefing
- Fallback: try to match to a market/contract, else helpful response

Response formatting:
- SHORT. 3-5 lines for quick queries.
- Emoji: red circle high urgency, yellow circle medium, green circle calm, chart data, lightning breaking
- Bold key numbers and names with markdown
- End with suggested action or follow-up question when relevant
- Parse mode: Markdown for Telegram

Example:
User: "anything interesting?"
Pythia: "lightning **Fed rate cut** probability jumped 11pts in the last hour. Three signal layers lit up: prediction markets, congressional trading, Twitter velocity. Historical hit rate when 3 layers converge on fed_rate: 73%.

Defense names have not repriced yet. Want me to watch them?"

User: "yeah watch rtx"
Pythia: "Done. Watching RTX. I will ping you if it moves >1% or if the fed signal develops."

User: "how have you been doing lately"
Pythia: "chart Last 30 days: 47 confluence alerts. 34 correct (72%). Best: fed_rate (81%). Worst: crypto_regulation (58%). Avg lead time: 4.2hrs."

User: "whats the vibe"
Pythia: "yellow circle **Policy Uncertainty** regime. Fed + tariffs + recession clusters all active. Last similar pattern: Sep 2025, SPX -2.3% in 48h."

### 2. src/pythia_live/bot.py — Telegram Bot Runner

Async Telegram bot using python-telegram-bot>=21.0:
- PythiaBot class with Application builder
- handle_message: any text message goes to companion.respond()
- send_proactive: push alerts to users
- Commands as shortcuts only: /start, /watchlist, /proof, /settings
- Primary interface is natural text, not slash commands
- reply_text with parse_mode Markdown

### 3. src/pythia_live/user_context.py — Per-User Memory

- UserContext class storing per-user state in JSON files (data/users/{user_id}.json)
- get_context: preferences, watchlist, last 10 messages, active watches
- update_context: store message+response for continuity
- add_watch / remove_watch: manage asset watches with thresholds
- get_watches: active watches for a user

### 4. run_bot.py (root level)

Simple runner:
- Loads PYTHIA_TELEGRAM_TOKEN from env
- Creates PythiaCompanion + PythiaBot
- Runs the bot

### 5. Update requirements.txt
Add: python-telegram-bot>=21.0

## Design Rules
- NO external LLM calls. All rule-based with dynamic data. Fast, free, predictable.
- Async throughout
- Response latency under 2 seconds
- Full type hints and docstrings
- Follow existing code patterns
- Handle errors gracefully, never show tracebacks
- Do NOT modify existing files, only create new ones
